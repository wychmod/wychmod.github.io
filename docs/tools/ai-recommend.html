📜 人工智能发展简史

里程碑

按时间向下梳理 AI 从理论、深度学习到大模型与智能体的关键节点；近年条目重点补充主流代表模型的发布时间和核心特性，包含中国模型。

1950
🧪
图灵测试诞生
艾伦·图灵发表《计算机器与智能》论文，提出著名的"图灵测试"，为人工智能研究奠定了理论基础，被誉为"人工智能之父"。
1956
🎓
AI正式诞生
达特茅斯会议召开，约翰·麦卡锡首次提出"人工智能"(Artificial Intelligence)术语，标志着AI作为独立学科正式诞生。
1966
💬
ELIZA聊天机器人
MIT的约瑟夫·维森鲍姆创造了ELIZA，这是历史上第一个聊天机器人，能够模拟心理治疗师与人对话，开创了人机对话的先河。
1997
♟️
深蓝战胜棋王
IBM的深蓝(Deep Blue)超级计算机击败国际象棋世界冠军卡斯帕罗夫，首次证明AI在特定领域可以超越人类顶尖水平。
2012
🧠
深度学习突破
AlexNet在ImageNet竞赛中大幅领先，深度学习迎来爆发。GPU加速和大数据时代的到来，推动AI进入新纪元。
2016
🎮
AlphaGo击败李世石
Google DeepMind的AlphaGo以4:1战胜围棋世界冠军李世石，展示了深度强化学习的强大能力，引发全球AI热潮。
2017
🔄
Transformer架构问世
Google发表《Attention Is All You Need》论文，提出Transformer架构，彻底改变了自然语言处理领域，为GPT等模型奠定基础。
2020.06
📚
GPT-3推动大模型范式
OpenAI发布1750亿参数的GPT-3，少样本学习、自然语言生成和通用任务迁移能力显著提升，预示“规模化预训练 + 提示词”成为主流范式。
2022.11
💥
ChatGPT引爆全球
ChatGPT上线，把对话式交互、指令跟随和RLHF带到大众视野，AI产品从研究演示进入日常办公、学习和开发工作流。
2023.03
🚀
GPT-4与国产对话模型启动
GPT-4发布，强化复杂推理、代码与图像输入能力；同月百度文心一言等国产对话产品亮相，中国大模型应用进入公开竞争阶段。
2023.12
💎
Gemini 1.0原生多模态
Google发布Gemini 1.0，强调文本、图像、音频、视频和代码的原生多模态理解，推动模型从“接入多模态”走向“原生多模态”。
2024.03
🧠
Claude 3系列发布
Anthropic发布Claude 3 Opus、Sonnet、Haiku，强化长上下文、视觉理解与安全对齐，形成“旗舰、均衡、轻量”的模型产品矩阵。
2024.05
🎙️
GPT-4o开启实时多模态
OpenAI发布GPT-4o，统一文本、图像和语音交互，显著降低实时语音对话延迟，AI助手开始更接近自然的人机协作界面。
2025.01
🔬
DeepSeek-R1开源推理模型
DeepSeek发布R1，强化数学、代码和逻辑推理，并开源模型权重与蒸馏版本，推动低成本高性能推理模型成为全球焦点。
2025.03
🧭
混合推理与国产多模态加速
  • Claude 3.7 Sonnet：混合推理模型，可在快速响应与延长思考之间切换。
  • Gemini 2.5 Pro：Google“thinking model”，强化复杂推理、科学、数学和代码。
  • 文心4.5 / X1：百度发布原生多模态基础模型与深度思考推理模型。
2025.04
🧱
开源与混合推理继续扩散
Meta发布Llama 4 Scout、Maverick，采用原生多模态和MoE架构；阿里发布Qwen3，支持“思考/非思考”混合模式、119种语言和更强Agent能力。
2025.05
🛠️
编程智能体成为主战场
Anthropic发布Claude Opus 4、Sonnet 4并强化Claude Code；OpenAI发布Codex云端编程智能体，AI开始从“补全代码”进入“理解仓库、执行任务、跑测试”的工程协作阶段。
2025.07
🇨🇳
中国模型进入Agent与代码竞争
  • Qwen3-Coder：480B MoE、原生256K上下文，可扩展到1M，面向Agentic Coding。
  • Kimi K2：月之暗面发布开源MoE模型，强调代码、工具使用和Agent任务。
  • GLM-4.5：智谱面向Agent、推理、代码三类任务发布ARC基础模型。
2025.08
⚙️
GPT-5统一模型体验
OpenAI发布GPT-5，强调更强推理、代码、写作和业务场景能力，并在ChatGPT中统一多条模型路线，减少用户手动选择模型的负担。
2025.09
⌨️
GPT-5-Codex与Qwen代码生态
OpenAI发布GPT-5-Codex系统卡，Codex进入面向软件工程的专用模型阶段；Qwen Code、Claude Code代理接入等工具让国产代码模型更容易进入现有开发流程。
2026.02
📈
Gemini 3.1 Pro强化复杂任务
Google发布Gemini 3.1 Pro，继续增强复杂问题求解、长任务规划和推理能力，模型竞争进一步转向“能否稳定完成复杂工作”。
2026.05
🌟
前沿模型进入高可靠工程阶段
Claude Opus 4.8等模型继续提升代码、知识工作和可靠性。行业重点从“模型会不会回答”转向“能否在工具、权限、测试和审计约束下稳定交付”。
