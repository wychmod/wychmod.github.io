📖 时间戳使用指南

💡 什么是时间戳？

时间戳（Timestamp）是指格林威治时间1970年01月01日00时00分00秒（北京时间1970年01月01日08时00分00秒）起至现在的总秒数或毫秒数。 它是一个用来表示时间的数字，在计算机系统中被广泛使用。

🎯 核心概念：
• 秒级时间戳：10位数字，精确到秒，如：1736559600
• 毫秒级时间戳：13位数字，精确到毫秒，如：1736559600000

🔧 使用场景

  • 数据存储：在数据库中存储时间，使用时间戳可以节省空间且便于计算
  • 接口开发：API接口传递时间参数时，时间戳是通用的格式
  • 日志记录：系统日志中使用时间戳记录事件发生的时间
  • 时间计算：计算时间差、时间比较等操作更加方便
  • 跨时区处理：时间戳是UTC时间，不受时区影响
  • 缓存过期：设置缓存的过期时间，判断数据是否过期

// 日期转时间戳

console.log(newTimestamp); // 1736559600000

Java 
import java.time.*;
import java.time.format.DateTimeFormatter;

// 获取当前时间戳（秒）
long timestamp = Instant.now().getEpochSecond();
System.out.println(timestamp); // 1736559600

// 获取当前时间戳（毫秒）
long timestampMs = System.currentTimeMillis();
System.out.println(timestampMs); // 1736559600000

// 时间戳转日期时间
LocalDateTime dateTime = LocalDateTime.ofEpochSecond(
    1736559600, 0, ZoneOffset.ofHours(8)
);
DateTimeFormatter formatter = DateTimeFormatter.ofPattern("yyyy-MM-dd HH:mm:ss");
System.out.println(dateTime.format(formatter)); // 2026-01-11 08:00:00

// 日期时间转时间戳
LocalDateTime dt = LocalDateTime.parse("2026-01-11 08:00:00", formatter);
long ts = dt.toEpochSecond(ZoneOffset.ofHours(8));
System.out.println(ts); // 1736559600
Go 
package main

import (
    "fmt"
    "time"
)

func main() {
    // 获取当前时间戳（秒）
    timestamp := time.Now().Unix()
    fmt.Println(timestamp) // 1736559600

    // 获取当前时间戳（毫秒）
    timestampMs := time.Now().UnixMilli()
    fmt.Println(timestampMs) // 1736559600000

    // 时间戳转日期时间
    t := time.Unix(1736559600, 0)
    fmt.Println(t.Format("2006-01-02 15:04:05")) // 2026-01-11 08:00:00

    // 日期时间转时间戳
    layout := "2006-01-02 15:04:05"
    dt, _ := time.Parse(layout, "2026-01-11 08:00:00")
    ts := dt.Unix()
    fmt.Println(ts) // 1736559600
}

⚠️ 注意事项

  • 时区问题：时间戳本身是UTC时间，转换为本地时间时需要考虑时区差异
  • 精度选择：根据实际需求选择秒级或毫秒级时间戳
  • 2038年问题：32位系统中时间戳最大值为2147483647（2038-01-19），超过会溢出
  • 闰秒处理：某些系统中需要特别处理闰秒的情况

🎓 扩展知识

为什么选择1970年1月1日作为起点？
这个时间点被称为"Unix纪元"（Unix Epoch），是由Unix系统的开发者在1970年代初期选定的。选择这个时间点主要是因为：

  • Unix系统开始开发的时间在1969年，选择1970年是一个近似的整数年份
  • 这个时间点足够早，可以覆盖几乎所有需要记录的现代计算机时间
  • 便于系统实现和计算，成为了事实上的标准
