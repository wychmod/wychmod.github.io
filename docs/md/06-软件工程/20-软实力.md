## 一、什么是软实力

技术能力让你**能做事**，软实力让你**做成事**。

| 类型 | 举例 |
|---|---|
| 沟通 | 跨团队协作、向上汇报 |
| 领导力 | 带新人、推动项目 |
| 时间管理 | 优先级判断、专注 |
| 学习 | 快速上手新技术 |
| 抗压 | 加班、紧急 bug |

## 二、技术人最该补的 3 个软实力

### 2.1 写清楚

> 💡 "写" 是技术人最被低估的软实力。

- **代码注释**：为什么这样做 > 做了什么
- **技术文档**：让别人 5 分钟能上手
- **周报 / 总结**：量化结果 + 反思
- **PPT / 方案**：让非技术人也能听懂

### 2.2 讲清楚

- **会议发言**：先说结论再说理由
- **Code Review**：对事不对人，给具体建议
- **汇报**：用"现状-问题-方案-风险"框架
- **讲故事**：把技术决策讲成有起承转合的故事

### 2.3 听懂别人

- **听懂业务**：知道为什么做 > 怎么做
- **听懂需求**：重复一遍确认（"所以您的意思是……"）
- **听懂情绪**：对方没说出口的担心

## 三、职场进阶关键节点

| 阶段 | 关键能力 |
|---|---|
| 初级（1-3 年） | 写好代码、独立 debug |
| 中级（3-5 年） | 独立负责模块、影响 5-10 人 |
| 高级（5-8 年） | 主导大项目、影响跨团队 |
| 资深（8+ 年） | 制定方向、影响行业 |
