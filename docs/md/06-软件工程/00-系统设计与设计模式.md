### 2.1 系统设计通用方法论（4 步法）

```
1. 需求澄清（Clarify）
   - 功能需求（核心功能）
   - 非功能需求（QPS / 存储 / 可用性 / 一致性）
   - 边界场景

2. 估算（Estimate）
   - QPS / DAU 推算
   - 存储量
   - 带宽

3. 设计（Design）
   - High-Level：画架构图（Client / API / Service / DB）
   - Data Model：核心表结构
   - API Design：RESTful 接口
   - 关键流程：读流程 / 写流程

4. 深入（Deep Dive）
   - 单点故障 → 多副本
   - 性能瓶颈 → 缓存 / 异步 / 分片
   - 数据一致性 → CAP / BASE / 共识算法
```

### 2.2 经典案例速记

#### TinyURL（短链系统）
- **生成方案**：Hash(id) / 自增 ID + Base62
- **写流程**：长链 → ID → 短码 → 写 DB + 缓存
- **读流程**：短码 → 查 DB → 302 跳转
- **关键**：Hash 冲突处理、缓存预热

#### News Feed（信息流）
- **推模型（Fanout on Write）**：发消息时写入所有粉丝收件箱
- **拉模型（Fanout on Read）**：读时聚合关注列表
- **混合**：明星用推，普通用户用拉
- **关键**：分页策略、热点用户限流

#### 限流（RateLimiter）
- **固定窗口**：简单但有突刺问题
- **滑动窗口**：更平滑
- **令牌桶**：允许突发流量
- **漏桶**：强制匀速
