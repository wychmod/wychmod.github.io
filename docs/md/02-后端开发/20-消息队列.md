## 一、消息队列核心概念

| 概念 | 含义 |
|---|---|
| **Producer** | 消息生产者 |
| **Consumer** | 消息消费者 |
| **Broker** | 消息服务器 |
| **Topic / Queue** | 消息主题 / 队列 |
| **Partition** | 分区（Kafka） |
| **Offset** | 消费位置 |
| **Consumer Group** | 消费者组（Kafka） |
| **ACK** | 消息确认 |

## 二、MQ 三大作用

1. **解耦**：服务间不直接调用
2. **异步**：提高响应速度
3. **削峰**：缓冲突发流量

## 三、RabbitMQ

> 📚 [RabbitMQ.md](../../archive/old-mq-notes/RabbitMQ.md)（27KB，已归档）

### 3.1 特点

- **Erlang** 编写
- AMQP 协议
- 适合**中小规模**业务
- 灵活的路由（Exchange + Binding）

### 3.2 核心概念

| 概念 | 作用 |
|---|---|
| **Exchange** | 接收消息 + 路由 |
| **Queue** | 存储消息 |
| **Binding** | Exchange → Queue 的路由规则 |
| **Routing Key** | 路由键 |
| **VHost** | 虚拟主机（多租户隔离） |

### 3.3 Exchange 类型

| 类型 | 路由规则 |
|---|---|
| **direct** | 精确匹配 routing key |
| **topic** | 通配符匹配（`*` 一个词 / `#` 多个） |
| **fanout** | 广播（忽略 routing key） |
| **headers** | 根据 header 匹配（少用） |

## 四、RocketMQ

### 4.1 设计与原理

> 📚 [RocketMQ设计与原理.md](../../archive/old-mq-notes/RocketMQ设计与原理.md)（44KB，已归档）

- **Java** 编写（阿里开源）
- 适合**大规模**金融级业务
- 高吞吐、低延迟
- 严格的消息顺序

**核心组件**：
- NameServer（轻量级注册中心）
- Broker（消息存储）
- Producer / Consumer
- Topic + Queue（4 默认 + 8 默认读队列 = 8 读 + 4 写）

**消息存储**：
- CommitLog（顺序写）
- ConsumeQueue（消费队列）
- IndexFile（按 key / 时间查询）

## 五、Kafka

> 📚 [kafka基础知识.md](../../archive/old-mq-notes/kafka/kafka基础知识.md)（3KB 入门）

- **Scala** 编写（LinkedIn 开源）
- 适合**日志收集** + **流式处理**
- 极高的吞吐量（百万级 TPS）
- 严格的消息顺序（按 partition）

**核心概念**：
- Topic（主题）
- Partition（分区）
- Consumer Group（消费者组）
- Offset（消费位移）
- Broker（节点）

**典型场景**：
- 日志收集（Filebeat → Kafka → ELK）
- 事件溯源
- 实时数据管道
- 流处理（Kafka Streams / Flink）

## 七、MQ 对比与选型（2026）

| 维度 | RabbitMQ | RocketMQ | Kafka |
|---|---|---|---|
| 语言 | Erlang | Java | Scala/Java |
| 协议 | AMQP | 自研 | 自研 |
| 吞吐 | 万级 | 十万级 | **百万级** |
| 延迟 | 微秒 | 毫秒 | 毫秒 |
| 顺序 | 弱 | 强 | 分区内强 |
| 适用 | 中小业务、复杂路由 | 金融级、大规模 | 日志、流处理 |
| 云原生 | 一般 | 一般 | **强**（Kraft 模式无 ZK） |
| 运维 | 简单 | 中等 | 较复杂 |

**选型建议**：
- **简单业务 + 复杂路由** → RabbitMQ
- **金融级一致性 + 事务消息** → RocketMQ
- **日志 / 流处理 / 大数据** → Kafka
- **2026 新项目** → Kafka（Kraft 模式）或云托管（Pulsar / Confluent Cloud）
