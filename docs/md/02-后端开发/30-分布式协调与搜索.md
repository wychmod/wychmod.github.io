## 一、Zookeeper

### 1.1 基础

> 📚 [Zookeeper基础.md](../../archive/old-distributed-coord-notes/Zookeeper基础.md)（29KB，已归档）

**Zookeeper 是什么**：
- 分布式协调服务
- 提供：配置管理 / 命名服务 / 分布式锁 / 集群管理
- 基于 **ZAB 协议**（类似 Paxos）
- 树形 ZNode 结构

**核心特性**：
- **顺序一致性**：按发送顺序应用
- **原子性**：事务要么全成要么全败
- **单一视图**：客户端看到的服务端视图一致
- **可靠性**：消息投递保证
- **实时性**：一段时间内客户端能看到最新数据

**ZNode 类型**：
- 持久节点（PERSISTENT）
- 临时节点（EPHEMERAL）：会话结束删除
- 顺序节点（SEQUENTIAL）
- 临时顺序节点

**Watcher 机制**：事件通知（一次性）

### 1.3 典型应用

- **Hadoop YARN**：集群管理
- **HBase**：元数据
- **Kafka**：早期版本依赖 ZK 做 Controller 选举
- **Dubbo**：服务注册与发现
- **分布式锁**：临时顺序节点

## 二、Elasticsearch

### 2.1 分布式搜索引擎

> 📚 [elasticsearch分布式搜索引擎.md](../../archive/old-distributed-coord-notes/elasticsearch分布式搜索引擎.md)（**119KB** 完整版，已归档）

**ES 是什么**：
- 基于 **Lucene** 的分布式搜索和分析引擎
- 倒排索引（inverted index）
- 近实时（NRT）搜索
- RESTful API

**核心概念**：
- **Index**（索引）：一组相似文档
- **Document**（文档）：基本单位，JSON
- **Mapping**（映射）：字段类型定义
- **Shard**（分片）：Index 的水平切分
- **Replica**（副本）：分片的备份
- **Node**（节点）：ES 实例
- **Cluster**（集群）：多个 Node 组成

**集群角色**：
- Master（主节点）
- Data Node（数据节点）
- Coordinating Node（协调节点）
- Ingest Node（预处理节点）

**写入流程**：
1. 协调节点接收请求
2. 路由到对应分片
3. 写入 Translog + Memory Buffer
4. Refresh 到 Segment（默认 1s）
5. Flush 到磁盘

**查询流程**：
1. 协调节点接收
2. 广播到各分片
3. 各分片本地查询
4. 合并 + 排序 + 返回

```bash
docker compose -f docker-compose.yml up -d
```
