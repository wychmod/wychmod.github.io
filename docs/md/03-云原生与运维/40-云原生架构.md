## 一、什么是云原生

> 📚 完整定义见 [云原生应用架构.md](../../archive/old-cloud-native-notes/云原生应用架构.md)（933B，已归档）

**CNCF（Cloud Native Computing Foundation）定义**：
> 云原生技术使组织能够在**公有云、私有云和混合云**等现代动态环境中，构建和运行**可弹性扩展**的应用。
> 代表技术包括：**容器、服务网格、微服务、不可变基础设施和声明式 API**。

## 二、四大核心特征

| 特征 | 含义 |
|---|---|
| **容器化** | Docker / containerd，标准化打包 |
| **编排** | Kubernetes，自动调度 + 扩缩 |
| **微服务** | 小而自治的服务，独立部署 |
| **声明式 API** | "我想要什么"而非"怎么实现" |

## 三、12 因素应用（12-Factor App）

> 💡 云原生应用的设计原则

| # | 因素 | 说明 |
|---|---|---|
| 1 | 基准代码 | 一份代码库，多处部署 |
| 2 | 依赖 | 显式声明依赖（package.json/pom.xml） |
| 3 | 配置 | 配置存在**环境变量**中 |
| 4 | 后端服务 | 把 DB / 缓存 / 消息队列当作**附加资源** |
| 5 | 构建/发布/运行 | 严格分离构建和运行 |
| 6 | 进程 | 应用以**无状态进程**运行 |
| 7 | 端口绑定 | 通过**端口绑定**提供服务 |
| 8 | 并发 | 通过**进程模型**横向扩展 |
| 9 | 易处理 | 快速启动 + 优雅终止 |
| 10 | 开发/生产等价 | 尽量保持环境一致 |
| 11 | 日志 | 把日志当作**事件流** |
| 12 | 管理进程 | 一次性管理任务作为独立进程 |

## 四、典型云原生技术栈

```
┌─────────────────────────────────────┐
│  应用层（业务代码）                  │
├─────────────────────────────────────┤
│  微服务框架（Spring Cloud / gRPC）   │
├─────────────────────────────────────┤
│  服务网格（Istio / Linkerd）         │
├─────────────────────────────────────┤
│  容器（Docker / containerd）         │
├─────────────────────────────────────┤
│  编排（Kubernetes）                  │
├─────────────────────────────────────┤
│  可观测（Prometheus / Grafana / Loki）│
├─────────────────────────────────────┤
│  CI/CD（GitHub Actions / Argo CD）   │
└─────────────────────────────────────┘
```

## 五、2026 年云原生现状

> ⚠️ 原文件 933B 是 2018 年内容，**强烈过时**

| 维度 | 2018 | 2026 |
|---|---|---|
| 容器运行时 | Docker | containerd / CRI-O |
| 编排 | K8s 新生 | K8s 标准，云厂商托管（EKS/AKS/GKE）|
| 服务网格 | Istio 新生 | Istio 成熟 + Linkerd + Cilium |
| 配置 | ConfigMap | ConfigMap + External Secrets + Vault |
| 镜像 | Docker Hub | 阿里云 ACR / 腾讯云 TCR / 自建 Harbor |
| GitOps | Argo CD 新生 | Argo CD / Flux 标准 |
| 可观测 | Prometheus + Grafana | + Loki + Tempo + OpenTelemetry |

## 六、学习路径建议

```
1. Docker 基础（容器/镜像/网络/卷）
   ↓
2. Kubernetes 基础（Pod/Deployment/Service/Ingress）
   ↓
3. Helm 包管理
   ↓
4. CI/CD（GitHub Actions 或 GitLab CI）
   ↓
5. 服务网格（Istio 基础）
   ↓
6. 可观测（Prometheus + Grafana）
   ↓
7. GitOps（Argo CD）
```
