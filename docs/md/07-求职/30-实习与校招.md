## 一、实习面试核心考点

### 1.1 网络（高频 4 题）

| # | 题目 | 一句话 |
|---|---|---|
| 1 | TCP 三次握手？ | SYN → SYN+ACK → ACK，确保双方收发能力正常 |
| 2 | TCP 四次挥手？ | 主动方 FIN → ACK → 被动方 FIN → ACK |
| 3 | TCP 拥塞控制？ | 慢启动 + 拥塞避免 + 快重传 + 快恢复 |
| 4 | TCP vs UDP？ | TCP 可靠但慢（握手/重传/排序），UDP 快但不保证 |

### 1.2 MySQL（高频 5 题）

| # | 题目 | 一句话 |
|---|---|---|
| 1 | ACID？ | 原子性/一致性/隔离性/持久性 |
| 2 | 事务隔离级别？ | 读未提交/读已提交/可重复读/串行化 |
| 3 | 索引失效场景？ | 函数/类型转换/前导模糊/不满足最左前缀 |
| 4 | MySQL 锁类型？ | 行锁/表锁/意向锁/Gap 锁 |
| 5 | MVCC 原理？ | 多版本并发控制，通过 undo log 实现非阻塞读 |

### 1.3 Redis（高频 3 题）

| # | 题目 | 一句话 |
|---|---|---|
| 1 | 数据结构？ | SDS / 字典 / 跳跃表 / 压缩列表 |
| 2 | 持久化？ | RDB（快照）+ AOF（日志追加） |
| 3 | 集群方案？ | 主从 + 哨兵 + Cluster |

### 1.4 多线程 / 并发

- 线程 vs 进程
- 线程池 7 个参数
- synchronized / volatile / CAS
- 死锁 4 条件 + 解决方案

### 1.5 Linux 常用

- 查看进程 / 端口 / 内存
- grep / awk / sed 文本三剑客
- 文件权限 / 用户管理

## 二、笔试技巧

### 2.1 算法题

```
1. 看题 5 分钟，列思路（不要上来就写代码）
2. 写代码 10-15 分钟
3. 测试样例 5 分钟（边界情况）
4. 优化时间复杂度（如果可能）
```

**常见题型**：
- 数组 / 字符串操作
- 链表（反转 / 合并 / 环检测）
- 二叉树（遍历 / BFS / DFS）
- 动态规划（背包 / 最长子序列）
- 图（BFS / DFS / 最短路）

### 2.2 主观题

- **系统设计**：从需求 → 架构 → 选型 → 权衡
- **SQL 编写**：注意语法 + 边界 + 性能
- **代码 review**：找出 bug + 提出改进

## 四、实习/校招准备路线

```
基础（1-2月）
├── 计算机网络（TCP/HTTP 重点）
├── 操作系统（进程/线程/内存）
├── MySQL（索引/事务/锁）
├── Redis（数据结构/持久化）
└── 编程语言（Java/Python 之一）

项目（1月）
├── 1-2 个能讲清楚的项目
├── 每个项目准备 5-10 个深挖问题
└── 难点 / 亮点 / 量化结果

刷题（1-2月）
├── LeetCode 100-200 题（按类型）
├── 剑指 Offer 66 题
└── 系统设计（校招高阶岗位）

面试（持续）
├── 模拟面试（找同学/朋友）
├── 复盘每场面试（录音/记录）
└── 针对性补漏
```
