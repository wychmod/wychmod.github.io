# 面试方法论

> 通用面试准备：自我介绍、HR 提问、谈薪、反向提问、职业规划。
> 原文整合自 [`Java面试汇总.md`](../archive/old-interview-notes/Java面试汇总.md)（已归档）。

---

## 一、自我介绍

> 💡 1-2 分钟，**突出"和岗位的匹配度"**，不是流水账。

**结构**：
1. 背景（学校 / 专业 / 学历，1 句）
2. 核心技能（2-3 个 + 1 句成果）
3. 关键项目（1-2 个最有代表性的，STAR 法则）
4. 求职动机（为什么这家公司 / 这个岗位）
5. 长期规划（1 句）

**模板**（中文版，2 分钟）：

> 面试官您好，我叫 XXX，XX 大学 XX 专业硕士研究生。本科期间学习了 Java 基础和 Spring/Mybatis 框架，研究生阶段主要做后端开发，实习期间在公司参与了 SpringBoot + MyBatis + Redis 的项目，做过优惠券系统、抽奖系统等。
>
> 平时喜欢写技术笔记和博客，已经在 GitHub 上开源了一些项目，累计有几十个 star。希望能加入贵公司，继续在 XX 方向深耕。

> 💡 英文版只是把上面的中文翻译一遍，准备一份即可。

## 二、HR 常见问题

### 2.1 谈薪资

> 💡 不要先出价，先反问对方预算范围。

```
参考话术：
"我看招聘 JD 上写的是 15-25K，我的期望在 18K 附近，但更看重公司平台和发展空间。
请问这个职位的预算范围是怎样的？"
```

**反问 HR**（看诚意）：

1. 能否聊一下作为公司员工对公司的感受？
2. 用三个词形容您对公司的印象？
3. 如果是您，您会给自己的团队/领导什么评价？
4. 团队目前最大的挑战是什么？
5. 面试完多久能给回复？（如果 HR 没主动说）

### 2.2 反向提问（"你有什么想问的？"）

**好的问题**（体现深度）：

- 团队目前用的技术栈是？ → 了解工作内容
- 这个岗位 3-6 个月内最希望候选人完成什么？ → 了解期望
- 团队的 code review 流程是怎样的？ → 了解工程文化
- 加班/出差强度？ → 实际关心

**避免的问题**：

- ❌ "公司主要业务是什么？"（JD 上写了）
- ❌ "有没有培训？"（过于被动）
- ❌ "薪资多少？"（谈薪环节问，不是这里）

## 三、面试官常问的"软"问题

| 问题 | 思路 |
|---|---|
| 为什么离开上一家？ | 强调"发展空间"而非"负能量" |
| 你的优点/缺点？ | 优点 2 个 + 例子；缺点 1 个 + 改进方案 |
| 5 年职业规划？ | 1 年融入 + 3 年骨干 + 5 年独当一面 |
| 最有成就感的事？ | STAR 法则（情境/任务/行动/结果） |
| 最大的失败？ | 说过程、说反思、说学到什么 |
| 为什么选我们公司？ | 提前做功课：业务/技术/文化/融资 |
| 手上其他 offer？ | 诚实说，但要表达"最想来你们" |

## 四、面试流程节奏

```
简历初筛 → 一面（技术基础）→ 二面（项目深挖）→ 三面（架构/综合）→ HR 面 → offer
```

> 💡 每轮面试结束前**必问**：面试反馈大概什么时候出？避免"无尽等待"。

## 五、offer 决策框架

> 💡 拿到多个 offer 后，**不要只看月薪**。

| 维度 | 权重建议 |
|---|---|
| 总包（base + 股票 + 奖金） | 30% |
| 公司/团队（业务、技术、文化） | 25% |
| 成长空间（mentor、技术栈） | 20% |
| 工作强度（加班、出差） | 15% |
| 地理位置 | 10% |

> 📌 **红线**：996 但薪资高、平台好 → 短期可以，长期谨慎。
