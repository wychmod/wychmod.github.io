> - [`人工智能-奈学/11-深度学习-基本知识.md`](../archive/old-llm-notes/人工智能-奈学/11-深度学习-基本知识.md)

### 2.1 从 Transformer 到 GPT

现代大语言模型（LLM）的底座是 **Transformer**。与 RNN/LSTM 不同，Transformer 完全基于**自注意力机制（Self-Attention）**，通过 Query / Key / Value 计算序列中任意两个位置的相关性，从而：

- 实现训练并行化，显著缩短训练时间；
- 缓解长距离依赖问题；
- 为后续 BERT、GPT、T5 等模型奠定基础。

Transformer 的两种经典范式：

| 范式 | 代表模型 | 预训练任务 | 典型应用 |
|---|---|---|---|
| 编码器（Encoder） | BERT | Masked Language Model + NSP | 文本理解、分类、NER |
| 解码器（Decoder） | GPT 系列 | 自回归语言建模 | 文本生成、对话、代码 |
| 编码器-解码器 | T5 / BART | Span corruption / Denoising | 翻译、摘要、问答 |

GPT 系列采用**自回归（Autoregressive）**方式训练：给定前文，预测下一个词。这种训练方式使 GPT 在生成连贯文本方面表现突出。

### 2.2 大模型发展时间线

| 时间 | 里程碑 |
|---|---|
| 2017 | Transformer 论文《Attention Is All You Need》发表 |
| 2018 | GPT-1、BERT 问世，开启预训练时代 |
| 2019 | GPT-2（1.5B）、T5 发布 |
| 2020 | GPT-3（175B），涌现 In-Context Learning、Few-Shot 能力 |
| 2022 | ChatGPT / InstructGPT，引入 RLHF 对齐 |
| 2023 | GPT-4、Claude 2、LLaMA 2，多模态初现 |
| 2024 | GPT-4o、Claude 3.5、Gemini 1.5，长上下文与多模态成标配 |
| 2025-2026 | GPT-5、Claude 4、DeepSeek V3 / R2，Agent 与推理能力成为主战场 |

### 2.3 关键能力

- **In-Context Learning**：不更新参数，仅通过在提示中给出示例即可让模型完成新任务。
- **Few-Shot / Zero-Shot**：用极少样本或零样本完成分类、翻译、抽取等任务。
- **指令遵循（Instruction Following）**：通过 SFT（监督微调）让模型按人类意图执行命令。
- **代码生成与推理**：GPT-3.5 之后通过代码数据训练显著增强了代码理解和复杂推理能力。
- **长上下文**：现代模型普遍支持 128K 甚至 1M+ token，RoPE、YaRN、Sliding Window Attention 是主流技术。

### 2.4 训练三阶段

1. **预训练（Pre-training）**：在海量无标注文本上做无监督语言建模，学习语言知识与世界常识。
2. **监督微调（SFT）**：用高质量指令数据微调，使模型学会按对话格式响应。
3. **对齐（RLHF / DPO）**：通过人类反馈或偏好对比，让输出更安全、有用、符合人类价值观。

> 💡 补充：2025-2026 年，RLHF 的替代方案 DPO（Direct Preference Optimization）和 KTO 因训练更稳定而越来越普及；同时出现了“推理时缩放（Inference-Time Scaling）”新范式，如 o1 / DeepSeek-R1 通过内部 Chain-of-Thought 在推理阶段消耗更多计算以提升答案质量。

### 2.5 基础 API 调用示例

```python
from openai import OpenAI

client = OpenAI()

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": "你是一位有帮助的编程助手。"},
        {"role": "user", "content": "用 Python 写一个快速排序。"},
    ],
    temperature=0.3,
    max_tokens=1024,
)

print(response.choices[0].message.content)
```

## 三、Prompt 工程

> 📚 原始资料：
> - [`gpt大模型应用/1-大模型基础：理论与技术的演进.md`](../archive/old-llm-notes/gpt大模型应用/1-大模型基础：理论与技术的演进.md)（提示学习、CoT、自洽性、思维树）

### 3.1 什么是 Prompt 工程

Prompt 工程是通过设计和优化输入提示，激发预训练大模型已有能力、改善输出质量的技术。核心思路是：**微调主要是引导模型使用已有知识，而提示工程是最高效的“能力激发”手段**。

### 3.2 基础技巧

| 技巧 | 说明 | 示例 |
|---|---|---|
| 角色设定 | 给模型一个明确角色，稳定输出风格 | “你是一位资深 Python 工程师” |
| 任务描述 | 清晰描述输入、输出格式与约束 | “请输出 JSON，字段为 title、summary” |
| Few-Shot | 提供 1-5 个输入输出示例 | 给出翻译示例后再给新句子 |
| 分隔符 | 用 ```、###、<> 等区分指令与内容 | 避免模型混淆 |
| 温度控制 | 低 temperature 适合确定性任务，高 temperature 适合创意 | 代码生成用 0.1-0.3 |

### 3.3 Chain-of-Thought（思维链）

CoT 通过在提示中要求模型“一步一步思考”，将复杂问题分解为中间推理步骤。关键发现：

- 对**小模型**可能无效甚至负面；
- 对**大模型**能显著涌现推理能力提升；
- 对**复杂问题**收益最大。

经典触发词：**"Let's think step by step."** / **"请逐步思考。"**

```python
prompt = """
问题：一个农场有鸡和兔共 35 只，脚共 94 只。鸡和兔各多少只？
请一步一步推理，并给出最终答案。
"""
```

### 3.4 自洽性（Self-Consistency）

对同一问题采样多条推理路径，然后投票或取多数答案。可显著提升数学、常识推理的准确率。

### 3.5 Tree-of-Thoughts（思维树）

ToT 把推理过程建模为树：根节点是初始问题，枝干是不同推理路径，叶子节点是候选答案，通过 BFS / DFS + 评估函数选择最优路径。适合需要多步探索的复杂任务（如数学证明、规划、游戏）。

### 3.6 结构化输出示例

```python
prompt = """
请从以下用户评价中提取实体和情感，输出 JSON：
{"entities": [{"name": "...", "sentiment": "positive|negative|neutral"}], "overall": "..."}

用户评价：这家餐厅的服务很好，但上菜太慢，总体体验一般。
"""

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": prompt}],
    response_format={"type": "json_object"},
)
```

> 💡 补充：2025-2026 年，Prompt 工程已发展为 **Prompt 编排（Prompt Orchestration）**，与 Function Calling、工具调用、记忆模块结合，形成动态提示模板系统。生产环境中建议使用版本化管理（如 LangChain Prompt Templates、PromptLayer）并做 A/B 测试。

## 四、Embedding 与向量检索

> 📚 原始资料：
> - [`gpt大模型应用/2-大模型开发基础：OpenAI Embedding.md`](../archive/old-llm-notes/gpt大模型应用/2-大模型开发基础：OpenAI%20Embedding.md)

> - [`人工智能-奈学/12-深度学习-开始使用Tensorflow.md`](../archive/old-llm-notes/人工智能-奈学/12-深度学习-开始使用Tensorflow.md)

### 4.1 从词袋到词嵌入

早期 NLP 使用**词袋（Bag-of-Words）**和 **TF-IDF** 表示文本，存在三个核心问题：

1. **词序不变性**："猫坐在垫子上" 和 "垫子坐在猫上" 得到相同向量；
2. **缺乏语义泛化**：cat、dog、goat 在独热编码中彼此无关；
3. **稀疏性高**：大型语料库维度爆炸，容易过拟合。

**词嵌入（Word Embedding）** 通过分布式表示将词映射到低维稠密向量，使得语义相近的词在向量空间中距离更近。经典例子：

```
vector("国王") - vector("男人") + vector("女人") ≈ vector("女王")
```

### 4.2 Word2Vec 的两种架构

| 架构 | 输入 | 输出 | 特点 |
|---|---|---|---|
| Skip-Gram | 目标词 | 上下文词 | 适合小语料、罕见词 |
| CBoW | 上下文词 | 目标词 | 训练快、对常见词效果好 |

训练时使用负采样（Negative Sampling），让模型区分真实词对与随机词对，从而学习语义关系。

### 4.3 现代 Embedding 模型

| 模型 | 特点 | 适用场景 |
|---|---|---|
| OpenAI `text-embedding-3-small/large` | 多语言、维度可压缩 | 通用语义检索 |
| BGE（BAAI） | 中文优化、开源 | 中文 RAG、问答 |
| M3E | 轻量、中文友好 | 中小型项目 |
| Cohere embed-multilingual-v3 | 多语言、长文本 | 跨语言检索 |
| Qwen Embedding | 阿里开源、中文强 | 中文知识库 |

### 4.4 Embedding API 调用示例

```python
from openai import OpenAI

client = OpenAI()

def get_embedding(text, model="text-embedding-3-small"):
    text = text.replace("\n", " ")
    return client.embeddings.create(
        input=[text], model=model
    ).data[0].embedding

vec = get_embedding("你好世界")
print(len(vec))  # 1536
```

### 4.5 向量相似度计算

```python
import numpy as np

def cosine_similarity(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

v1 = get_embedding("机器学习")
v2 = get_embedding("深度学习")
v3 = get_embedding("披萨")

print(cosine_similarity(v1, v2))  # 高
print(cosine_similarity(v1, v3))  # 低
```

### 4.6 向量数据库选型

| 数据库 | 特点 |
|---|---|
| Chroma | 轻量、易集成、适合原型 |
| FAISS | Meta 开源、高性能、本地优先 |
| Milvus / Zilliz | 企业级、分布式、大规模 |
| Pinecone | 托管服务、开箱即用 |
| pgvector | PostgreSQL 插件、与业务库统一 |

> 💡 补充：生产级 RAG 中，Embedding 质量往往比 LLM 选择更重要。建议对文档做**分块（Chunking）**、**去重**、**元数据标注**，并对不同字段（标题、正文、摘要）使用不同权重检索。

## 五、RAG 架构与实践

> 📚 原始资料：
> - [`gpt大模型应用/2-大模型开发基础：OpenAI Embedding.md`](../archive/old-llm-notes/gpt大模型应用/2-大模型开发基础：OpenAI%20Embedding.md)
> - [`人工智能-奈学/16-深度学习-自然语言处理nlp.md`](../archive/old-llm-notes/人工智能-奈学/16-深度学习-自然语言处理nlp.md)

### 5.1 为什么需要 RAG

大模型存在以下局限：

- **知识截止**：无法获取训练数据之后的信息；
- **幻觉（Hallucination）**：会生成看似合理但实际错误的内容；
- **领域知识弱**：对私有、专业、实时数据不了解；
- **成本高**：长上下文与频繁微调成本高昂。

**RAG（Retrieval-Augmented Generation，检索增强生成）** 通过在生成前从外部知识库检索相关上下文，把“参数化知识”和“非参数化知识”结合，显著缓解上述问题。

### 5.2 RAG 核心流程

```
用户提问
   ↓
Query 改写 / 扩展（可选）
   ↓
Embedding 编码 → 向量检索 Top-K
   ↓
重排序（Re-rank，可选）
   ↓
拼接上下文 + 原始问题 → Prompt
   ↓
LLM 生成答案
   ↓
引用溯源（Citation，可选）
```

### 5.3 关键模块

| 模块 | 作用 | 常用方案 |
|---|---|---|
| 文档加载 | 解析 PDF / Word / Markdown / 网页 | LangChain Document Loaders、Unstructured |
| 文本分块 | 控制上下文长度与语义完整性 | 固定长度、递归、语义分块、Agentic Chunking |
| Embedding | 将文本编码为向量 | OpenAI、BGE、M3E、Qwen |
| 向量存储 | 持久化与检索 | Chroma、FAISS、Milvus、pgvector |
| 检索 | 相似度搜索 | ANN、Hybrid Search（向量 + 关键词） |
| 重排序 | 提升 Top-K 质量 | Cohere Rerank、Cross-Encoder、BGE-Reranker |
| 生成 | 基于检索上下文回答 | GPT-4o、Claude 3.5、DeepSeek-V3、Qwen2.5 |

### 5.4 最小可用 RAG 示例

```python
from openai import OpenAI
import numpy as np

client = OpenAI()

documents = [
    "OpenAI 于 2022 年 11 月发布 ChatGPT。",
    "Transformer 论文发表于 2017 年，标题是 Attention Is All You Need。",
    "Python 是一种广泛使用的高级编程语言。",
]

# 1. 编码文档
def embed(text):
    return client.embeddings.create(
        input=[text], model="text-embedding-3-small"
    ).data[0].embedding

doc_embeddings = [embed(d) for d in documents]

# 2. 检索
def retrieve(query, top_k=2):
    q_vec = embed(query)
    scores = [np.dot(q_vec, d) for d in doc_embeddings]
    indices = np.argsort(scores)[::-1][:top_k]
    return [documents[i] for i in indices]

# 3. 生成
query = "ChatGPT 是什么时候发布的？"
context = "\n".join(retrieve(query))

prompt = f"""基于以下上下文回答问题：
{context}

问题：{query}
"""

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[{"role": "user", "content": prompt}],
)
print(response.choices[0].message.content)
```

### 5.5 进阶 RAG 模式

- **Self-RAG**：模型在生成过程中自行判断是否需要检索、检索结果是否相关、是否需要继续检索。
- **Corrective RAG**：当检索结果质量低时，自动切换为网络搜索或改写查询。
- **GraphRAG**：先构建知识图谱，再结合图遍历与向量检索，适合复杂关系型知识库。
- **Multi-Agent RAG**：由不同 Agent 分别负责检索、总结、验证、生成。

> 💡 补充：2025-2026 年，RAG 已从“向量检索 + LLM”进化为**多路召回 + 重排序 + 结构化引用 + 生成后校验**的完整流水线。评估指标除准确率外，还应关注**上下文召回率（Context Recall）**、**答案忠实度（Faithfulness）**和**引用正确性**。

## 六、Agent 与大模型结合

> 📚 原始资料：
> - [`动手学深度学习/动深-1-前言.md`](../archive/old-llm-notes/动手学深度学习/动深-1-前言.md)（强化学习、Agent 与环境交互）
> - [`人工智能-奈学/15-深度学习-循环神经网络.md`](../archive/old-llm-notes/人工智能-奈学/15-深度学习-循环神经网络.md)

### 6.1 从强化学习到 LLM Agent

强化学习中的 Agent 通过与环境交互，基于观察（Observation）、动作（Action）、奖励（Reward）学习策略（Policy），并在探索与利用之间权衡。LLM Agent 把大语言模型作为“大脑”，在文本环境中完成观察理解、规划、工具调用与行动执行。

### 6.2 LLM Agent 的核心组件

| 组件 | 作用 |
|---|---|
| **规划（Planning）** | 将复杂任务拆分为子任务，制定执行计划 |
| **记忆（Memory）** | 短期记忆（对话历史）+ 长期记忆（向量数据库） |
| **工具（Tools）** | 调用外部 API、数据库、搜索引擎、代码解释器 |
| **行动（Action）** | 执行工具调用并观察结果，迭代至目标完成 |

### 6.3 典型 Agent 设计模式

| 模式 | 说明 | 代表框架 |
|---|---|---|
| ReAct | 推理（Reasoning）与行动（Acting）交替进行 | LangChain、LlamaIndex |
| Plan-and-Solve | 先制定计划，再逐步执行 | LangGraph、AutoGen |
| Multi-Agent | 多个 Agent 分工协作，如编码、测试、评审 | AutoGen、CrewAI、MetaGPT |
| Reflection | Agent 自我反思、修正错误 | Reflexion、Self-Refine |

### 6.4 Function Calling 示例

```python
from openai import OpenAI

client = OpenAI()

tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "获取指定城市的当前天气",
            "parameters": {
                "type": "object",
                "properties": {
                    "city": {"type": "string", "description": "城市名称"},
                },
                "required": ["city"],
            },
        },
    }
]

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[{"role": "user", "content": "北京今天天气怎么样？"}],
    tools=tools,
)

print(response.choices[0].message.tool_calls)
```

### 6.5 最小 Agent 循环示例

```python
def agent_loop(query, max_steps=5):
    messages = [{"role": "user", "content": query}]
    for _ in range(max_steps):
        response = client.chat.completions.create(
            model="gpt-4o", messages=messages, tools=tools
        )
        msg = response.choices[0].message
        messages.append(msg)

        if not msg.tool_calls:
            return msg.content

        # 执行工具并返回结果
        for call in msg.tool_calls:
            result = execute_tool(call.function.name, call.function.arguments)
            messages.append({
                "role": "tool",
                "tool_call_id": call.id,
                "content": result,
            })
    return "达到最大步数"
```

> 💡 补充：2025-2026 年，Agent 框架正在从“单 Agent 调用工具”向**多 Agent 协作 + 状态机编排 + 人机协同**演进。LangGraph 凭借图结构状态机成为复杂 Agent 的首选；MCP（Model Context Protocol）则正在统一 Agent 与外部工具、数据源之间的接口标准。

### 7.1 卷积神经网络（CNN）基础

CNN 是视觉模型的核心，其关键操作包括：

- **卷积层**：通过卷积核提取局部空间特征，如边缘、纹理、形状；
- **激活函数**：ReLU 等引入非线性；
- **池化层**：Max/Avg Pooling 降低维度、增强平移不变性；
- **全连接层**：将特征图展平后进行分类。

经典网络演进：LeNet → AlexNet → VGG → ResNet → EfficientNet → Vision Transformer。

### 7.2 关键概念

| 概念 | 说明 |
|---|---|
| 感受野（Receptive Field） | 输出特征图中某元素对应的输入区域大小 |
| 填充（Padding） | 在输入边缘补零，控制输出尺寸 |
| 步幅（Stride） | 卷积核滑动间隔，控制下采样程度 |
| 通道（Channel） | 彩色图像有 RGB 三通道，卷积可扩展多输入/输出通道 |
| 1×1 卷积 | 跨通道信息融合，常用于降维与瓶颈设计 |

### 7.3 计算机视觉任务

- **图像分类**：判断图像类别；
- **目标检测**：识别图像中多个目标并给出边界框（YOLO、Faster R-CNN）；
- **图像分割**：像素级分类（语义分割、实例分割）；
- **图像增广**：翻转、裁剪、颜色变换，提高泛化能力；
- **迁移学习 / 微调**：在 ImageNet 预训练模型基础上微调下游任务。

### 7.4 多模态大模型

现代多模态模型将文本、图像、音频、视频统一编码到共享语义空间，典型代表包括：

| 模型 | 能力 |
|---|---|
| GPT-4o / GPT-4V | 图像理解、图文生成、端到端多模态对话 |
| Claude 3.5 Sonnet | 高分辨率图像理解、图表、文档分析 |
| Gemini 1.5 Pro | 长视频理解、跨模态推理 |
| Qwen-VL / Qwen2-VL | 开源中文多模态模型，支持 grounding、OCR |
| DeepSeek-VL2 | 视觉语言理解、文档分析 |

### 7.5 多模态 API 示例

```python
import base64

with open("invoice.png", "rb") as f:
    image_b64 = base64.b64encode(f.read()).decode("utf-8")

response = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "请提取这张发票的金额、日期和购买方。"},
                {
                    "type": "image_url",
                    "image_url": {"url": f"data:image/png;base64,{image_b64}"},
                },
            ],
        }
    ],
)
print(response.choices[0].message.content)
```

> 💡 补充：2025-2026 年，多模态应用已从“看图说话”拓展到**文档智能（Document AI）**、**视频理解**、**具身智能（Embodied AI）**和**自动驾驶感知决策**。视觉 Encoder + LLM Decoder 的架构成为主流，同时出现了专门的多模态 Agent 用于处理复杂 UI、设计稿、医疗影像等场景。

## 九、常见坑与补充

### 9.1 大模型应用常见陷阱

| 陷阱 | 说明 | 应对 |
|---|---|---|
| 幻觉 | 模型生成不存在的信息 | RAG、事实核查、降低 temperature、要求引用 |
| 提示注入 | 用户通过提示绕过安全限制 | 输入过滤、输出护栏、权限最小化 |
| 长上下文失效 | 信息藏在长文档中间时被忽略 | 重排序、分块摘要、关键信息前置 |
| 评估困难 | 生成任务难以量化 | LLM-as-a-Judge、人工评估、端到端测试集 |
| 成本失控 | Token 消耗与 API 调用频繁 | 缓存、模型路由、批量处理、本地小模型 |
| 数据泄露 | 敏感数据上传给第三方 API | 私有化部署、数据脱敏、合规审查 |

### 9.2 工程落地 checklist

- [ ] 明确任务边界并选择模型形态（API / 私有化 / 端侧）。
- [ ] 设计高质量 Prompt，含角色、约束、示例与输出格式。
- [ ] 构建 RAG 知识库：分块、Embedding、向量库、重排序。
- [ ] 按需引入 Function Calling 与 Agent（记忆、规划、反思）。
- [ ] 建立评估与监控体系：准确性、延迟、成本、安全、反馈闭环。
